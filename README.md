# TK AI Generator — Техническое описание API и механизмы интеграции

> **Версия документа:** 1.1  
> **Дата:** Апрель 2026  
> **Назначение:** Документ описывает способы подключения внешних приложений к системе генерации технологических карт ТОиР оборудования.

---

## 1. Общая информация

**TK AI Generator** — веб-приложение на FastAPI для автоматизированного формирования технологических карт технического обслуживания и ремонта (ТОиР) с использованием языковых моделей (OpenRouter, HuggingFace).

### Архитектура
```
┌────────────────────┐    ┌─────────────────┐   ┌─────────────────┐
│  Внешнее приложение│──▶│ TK AI Generator │──▶│ LLM-провайдер   │
│  (клиент)          │◀──│ (FastAPI, 8000) │◀──│ (OpenRouter/    │
└────────────────────┘    └─────────────────┘   │  HuggingFace)   │
                                │               └─────────────────┘
                                ▼
                     ┌─────────────────────┐
                     │  Файловое хранилище │
                     │  (settings.json,    │
                     │   временные файлы)  │
                     └─────────────────────┘
```

### Базовые параметры
| Параметр | Значение |
|----------|----------|
| **Базовый URL** | `http://localhost:8000` (по умолчанию) |
| **Протокол** | HTTP/1.1, HTTPS (при настройке reverse proxy) |
| **Формат данных** | JSON, multipart/form-data |
| **Кодировка** | UTF-8 |
| **Таймаут запроса** | 120 секунд (настраивается) |
| **Повторы при ошибках** | 3 попытки с экспоненциальной задержкой |

## 2. Аутентификация и авторизация

На текущий момент **аутентификация не реализована**. Все эндпоинты доступны публично.

### Рекомендации для продакшена:
```python
# Пример добавления API-key аутентификации
from fastapi import Security, HTTPException
from fastapi.security import APIKeyHeader

API_KEY_HEADER = APIKeyHeader(name="X-API-Key", auto_error=False)

async def verify_api_key(api_key: str = Security(API_KEY_HEADER)):
    if api_key != os.getenv("EXPECTED_API_KEY"):
        raise HTTPException(status_code=403, detail="Invalid API key")
    return api_key

# Использование в роуте:
@app.post("/api/chat", dependencies=[Depends(verify_api_key)])
async def chat_endpoint(...):
    ...
```

**Ответ (200 OK):**
```json
{
  "provider": "openrouter",

  "max_tokens": 3000,
  "master_prompt": "Ты инженер, специалист по формированию..."

```

---

### 3.3 `POST /api/settings` — Обновление настроек
Обновляет конфигурацию приложения (сохраняет в `settings.json`).

**Запрос (application/json):**
```json
{
  "provider": "openrouter",
  "api_key": "ваш_ключ",
  "model": "anthropic/claude-3.5-sonnet",
  "max_tokens": 4000,
  "master_prompt": "Кастомный системный промпт..."
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `provider` | string | `"openrouter"` |
| `api_key` | string | Ключ доступа к LLM-провайдеру |
| `model` | string | Идентификатор модели (зависит от провайдера) |
| `max_tokens` | integer | Лимит токенов в ответе (500–16000) |
| `master_prompt` | string | Системный промпт для ИИ |

**Ответ (200 OK):**
```json
{ "status": "ok" }
```

### 3.4 `POST /api/chat` — Основной эндпоинт генерации
Генерирует технологическую карту на основе параметров оборудования и опционального технического паспорта.

**Content-Type:** `multipart/form-data`

| Поле | Тип | Обязательный | Описание |
|------|-----|--------------|----------|
| `message` | string | Да | Текстовый запрос/инструкция для ИИ |
| `model_name` | string | Да | Название модели оборудования (напр. `"БелАЗ 7547"`) |
| `equipment_class` | string | Нет | Класс оборудования (напр. `"Самосвалы"`) |
| `subclass` | string | Нет | Подкласс (напр. `"Карьерные"`) |
| `file` | file | Нет | Технический паспорт: `.pdf`, `.docx`, `.txt`, `.csv` |
| `model` | string | Нет | Переопределение модели из настроек (опционально) |
| `provider` | string | Нет | Переопределение провайдера из настроек (опционально) |

**Пример запроса (curl):**
```bash
curl -X POST http://localhost:8000/api/chat \
  -F "message=Сгенерируй карту ТО для двигателя" \
  -F "model_name=БелАЗ 7547" \
  -F "equipment_class=Самосвалы" \
  -F "subclass=Карьерные" \
  -F "file=@/path/to/manual.pdf"
```

**Ответ (200 OK):**
```json
{
  "text": "Краткое текстовое описание результата...",
  "table_rows": [
    [
      "Система смазки",
      "Картер",
      "Осмотр",
      "Визуальный осмотр картера...",
      "ТО-1",
      "4320",
      "2.0",
      "1",
      "Слесарь по ремонту автомобилей, 3 разряд",
      "2.0",
      "",
      "",
      "",
      "",
      "Каска защитная, 1 шт; Очки защитные, 1 шт",
      "Затормозить технику; Выполнять работы при неработающем двигателе"
    ]
  ],
  "xlsx_file": "UEsDBBQABgAIAAAAIQD...[base64-encoded XLSX]...==",
  "xlsx_filename": "ТК_БелАЗ 7547_Самосвалы.xlsx"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `text` | string | Текстовое резюме от ИИ |
| `table_rows` | array[] | Массив строк таблицы (16 колонок) |
| `xlsx_file` | string (base64) | Готовый XLSX-файл в кодировке base64 |
| `xlsx_filename` | string | Рекомендуемое имя файла для скачивания |

> **Важно:** Поле `xlsx_file` может быть `null`, если генерация таблицы не удалась.

## 4. Структура данных таблицы

Все строки таблицы должны содержать **16 значений** в следующем порядке:

| № | Столбец | Описание | Пример |
|---|---------|----------|--------|
| 1 | Элемент | Крупный узел | `Система смазки` |
| 2 | Подэлемент | Деталь узла | `Картер` |
| 3 | Наименование операции | Тип работы | `Осмотр` |
| 4 | Краткое содержание работ | Описание | `Визуальный осмотр на трещины` |
| 5 | Вид ТОиР | Тип обслуживания | `ТО-1`, `ТО-2`, `ТР` |
| 6 | Периодичность | Интервал в моточасах | `4320` |
| 7 | Норма времени, часов | Длительность операции | `2.0` |
| 8 | Количество исполнителей | Число работников | `1` |
| 9 | Профессия/Квалификация | Требования к персоналу | `Слесарь, 3 разряд` |
| 10 | Трудоёмкость, чел/час | Расчётное значение | `2.0` |
| 11 | Наименование ТМЦ | Материалы | *(пусто)* |
| 12 | Количество ТМЦ | Объём материалов | *(пусто)* |
| 13 | Единицы измерения ТМЦ | Ед. измерения | *(пусто)* |
| 14 | Наименование инструменты | Инструмент | *(пусто)* |
| 15 | Средства индивидуальной защиты | СИЗ | `Каска; Очки; Перчатки` |
| 16 | Требования по безопасности | Инструкции | `Затормозить технику` |

> Пустые значения обозначаются пустой строкой `""`. Разделитель в сыром ответе ИИ — символ `|`.

## 5. Механизмы интеграции

### Вариант 1: Прямые HTTP-запросы (REST)
Подходит для скриптов, микросервисов, backend-приложений.

**Пример на Python:**
```python
import requests
import base64

def generate_tk(model_name: str, equipment_class: str, message: str, file_path: str = None):
    url = "http://localhost:8000/api/chat"
    files = {}
    if file_path:
        files = {"file": open(file_path, "rb")}
    
    data = {
        "message": message,
        "model_name": model_name,
        "equipment_class": equipment_class,
        "subclass": "Карьерные"
    }
    
    response = requests.post(url, data=data, files=files)
    result = response.json()
    
    # Сохранение XLSX
    if result.get("xlsx_file"):
        with open(result["xlsx_filename"], "wb") as f:
            f.write(base64.b64decode(result["xlsx_file"]))
    
    return result
```

### Вариант 2: Интеграция через веб-интерфейс (iframe)
Встраивание UI в корпоративный портал:
```html
<iframe 
  src="http://tk-ai.local:8000"

```

### Вариант 3: WebSocket (перспектива)
На текущий момент не реализован, но может быть добавлен для стриминга ответа ИИ в реальном времени.

### Вариант 4: Webhook-уведомления (перспектива)

```json
  "callback_token": "secret"
}
```

## 6. Обработка ошибок

| Код | Описание | Действия |
|-----|----------|----------|
| `400` | Некорректные параметры / отсутствующий API-ключ | Проверить обязательные поля и настройки |
| `413` | Файл превышает лимит размера | Уменьшить размер файла (<10 МБ рекомендуется) |
| `422` | Ошибка валидации multipart-формы | Проверить названия полей и Content-Type |
| `500` | Ошибка провайдера LLM / внутренняя ошибка | Проверить логи, повторить запрос с экспоненциальной задержкой |
| `504` | Таймаут ответа от ИИ | Увеличить `max_tokens` или выбрать более быструю модель |

**Пример ответа с ошибкой:**
```json
{
  "detail": "API ключ не установлен в настройках"
}
```

## Приложение: Формат ответа ИИ

Системный промпт требует от модели строгого формата ответа:

```
[ТЕКСТ_ОТВЕТ]
Краткое описание результата для пользователя
[/ТЕКСТ_ОТВЕТ]

[ТАБЛИЦА]
Элемент|Подэлемент|Наименование операции|...|Требования по безопасности
Значение1|Значение2|Значение3|...|Значение16
[/ТАБЛИЦА]
```

Парсер на стороне сервера извлекает блоки по маркерам `[ТЕКСТ_ОТВЕТ]` и `[ТАБЛИЦА]`, разделяет строки таблицы по символу `|` и формирует итоговый JSON-ответ.
